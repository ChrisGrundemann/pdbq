Privacy Policy

Effective date: March 28, 2026  ·  Operated by Grundemann Technology Solutions (GTS)

Short version: pdbq does not collect personal data, does not use cookies, and does not track you. Your API keys stay in your browser. Your query text is sent to the backend to generate an answer and is not stored.

1. Who we are

pdbq is operated by Grundemann Technology Solutions (GTS). This policy describes what data is collected when you use pdbq at peeringdb.ai or via the API at api.peeringdb.ai.

2. Data we collect

Server access logs — Our hosting provider (Fly.io) automatically records standard HTTP access logs for every request. These include:

  • Timestamp
  • HTTP method and path (e.g. POST /query)
  • Response status code
  • Source IP address

These logs are retained by Fly.io per their standard log retention policy. We do not log query text, API keys, or any user-identifying information beyond what is present in standard access logs.

Query text — When you submit a query, the text of your question is sent to our backend and forwarded to Anthropic's API to generate an answer. Query text is not stored or logged by GTS. It is processed transiently and discarded after the response is returned.

API keys — Your pdbq access key and Anthropic API key (if provided) are stored only in your browser's localStorage. They are never stored on our servers. Your Anthropic key is transmitted directly to Anthropic's API and is not retained by GTS.

3. Data we do not collect

  • No cookies of any kind
  • No analytics or tracking scripts
  • No account registration or personal profile data
  • No payment information

4. Third-party services

Anthropic — Query text is sent to Anthropic's API to generate answers. Anthropic's own privacy policy governs how they handle this data. See anthropic.com/privacy.

Fly.io — pdbq is hosted on Fly.io. Fly.io may retain server access logs per their own data retention policies. See fly.io/legal/privacy-policy.

Google Fonts — The web interface loads fonts from Google Fonts. This may result in your IP address being sent to Google's servers. See policies.google.com/privacy.

PeeringDB — The data returned by pdbq originates from PeeringDB. pdbq is not affiliated with PeeringDB. PeeringDB's own privacy policy applies to their platform.

5. Data retention

GTS does not maintain any user database or persistent user records. Server access logs are retained per Fly.io's standard policies, which we do not control.

6. Your rights

Since we do not collect or store personal data beyond standard server logs, there is no user data for GTS to provide, correct, or delete. If you have concerns about access log data, contact us and we will do our best to assist within what is technically possible.

7. Changes to this policy

We may update this policy as the service evolves. Material changes will be noted by updating the effective date above.

8. Contact

Questions? Open an issue at github.com/ChrisGrundemann/pdbq.

pdbq is an unofficial community tool. Not affiliated with PeeringDB.
